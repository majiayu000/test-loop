# Closed-Loop Drill Report — 2026-06-05

## Purpose

Verify the v0.2 scripts that test-loop imports from caff 0.1.4 still
work end-to-end after being relocated into a different repository, and
that the failure-classifier handles the real duplicate-line pattern
that [the caff 2026-06-02 drill](https://github.com/majiayu000/caff)
discovered.

This drill is a *synthetic* one: test-loop is a documentation-and-skills
repository, not a Swift project, so we cannot inject a real
Swift Testing failure. Instead we feed a hand-crafted log to
`bin/classify_failures.py` to exercise the same code path the real
drill exercises.

The two `error: ... not found` outcomes are expected. Both scripts

repository with no Swift code, so the Swift-specific default cannot
find anything. Cross-language defaults are a v0.2.1 follow-up.

## Step 4 output (the interesting one)

```json
{
  "failures": [
    "drillRejectsZero",
    "drillAssertsWrongValue"
  ],
  "failures_by_class": {
    "EXPECTED_FAILURE": 1,
    "ASSERTION_FAILURE": 1
  },
  "failures_grouped": {
    "EXPECTED_FAILURE": ["drillRejectsZero"],
    "ASSERTION_FAILURE": ["drillAssertsWrongValue"]
  }
}
```

Three things to note:

1. **Deduplication works.** The log emits two `✘` lines per failing
   test (one for the recorded issue, one for the failure summary);
   the classifier counts each failing test exactly once.
2. **Classification works.** `drillRejectsZero` matches the
   `EXPECTED_FAILURE` pattern (the `Rejects` keyword) and goes into
   its own bucket. `drillAssertsWrongValue` has no matching pattern
   and falls through to `ASSERTION_FAILURE` (the default bucket), as
   designed.
3. **`failures` is a list, not a set.** The two arrays are
   deduplicated but preserve first-seen order. This matters for any
   downstream tool that prints failing tests in the order they were
   observed.

## Reproducing this drill

```bash
git clone https://github.com/majiayu000/test-loop
cd test-loop

# Step 1: classifier self-test
python3 bin/classify_failures.py --self-test

# Step 2 & 3: expected errors (no swift source / no log yet)
bash bin/check_drift.sh         # exit 2
bash bin/render_report.sh --collect  # exit 2

# Step 4: synthetic drill
cat > /tmp/drill.log <<'EOF'
✔ Test somethingPassed() passed after 0.001 seconds.
✘ Test drillRejectsZero() recorded an issue at file:1:1.
✘ Test drillRejectsZero() failed after 0.001 seconds with 1 issue.
✘ Test drillAssertsWrongValue() recorded an issue at file:2:2.
✘ Test drillAssertsWrongValue() failed after 0.002 seconds with 1 issue.
EOF
python3 bin/classify_failures.py --in /tmp/drill.log
```

If any of those steps produces a different outcome, the loop has
regressed since this commit (`ba48acb` + `057d216`).
